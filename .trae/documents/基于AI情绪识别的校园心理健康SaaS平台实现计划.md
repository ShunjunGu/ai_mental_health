# 基于AI情绪识别的校园心理健康SaaS平台实现计划

## 1. 项目概述
构建一个基于AI情绪识别的校园心理健康SaaS平台，包含预警分级、咨询服务和教师端决策支持功能，帮助学校及时发现和干预学生心理健康问题。

## 2. 技术架构

### 2.1 前端技术栈
- React 18 + TypeScript
- Vite 构建工具
- Ant Design UI组件库
- Redux Toolkit 状态管理
- Axios API请求
- Socket.io 实时通信

### 2.2 后端技术栈
- Node.js + Express + TypeScript
- MongoDB 数据库
- Mongoose ODM
- JWT 身份认证
- Socket.io 实时通信

### 2.3 AI情绪识别
- 集成第三方AI情绪识别API（如百度AI、腾讯AI或OpenAI API）
- 支持文本、语音、图像三种情绪识别方式

## 4. 核心功能模块

### 4.1 用户管理模块
- 学生、教师、心理咨询师、管理员四种角色
- 基于角色的访问控制（RBAC）
- 用户注册、登录、密码重置功能

### 4.2 情绪识别模块
- 文本情绪识别（日记、留言等）
- 语音情绪识别（语音录音）
- 图像情绪识别（面部表情）
- 情绪识别历史数据存储

### 4.3 预警分级模块
- 基于情绪得分和持续时间的预警规则
- 四级预警分级（正常、轻度、中度、重度）
- 自动生成预警报告
- 多渠道预警通知（系统消息、邮件、短信）

### 4.4 咨询服务模块
- 在线预约心理咨询
- 实时咨询会话（文字、语音）
- 咨询记录管理
- 咨询效果评估

### 4.5 教师端决策支持模块
- 学生心理健康状况概览仪表盘
- 预警信息筛选和查看
- 智能干预建议生成
- 班级/年级心理健康统计报表

### 4.6 数据分析模块
- 学生情绪变化趋势分析
- 心理健康状况分布统计
- 预警事件统计分析
- 咨询服务效果评估

## 5. 数据库设计

### 5.1 用户表（Users）
- 基本信息（姓名、性别、年龄、学号/工号）
- 角色信息
- 登录凭证
- 创建时间和更新时间

### 5.2 情绪记录表（EmotionRecords）
- 用户ID
- 识别类型（文本/语音/图像）
- 原始内容/文件路径
- 情绪识别结果（情绪类型、得分）
- 识别时间

### 5.3 预警信息表（Alerts）
- 学生ID
- 预警级别
- 预警原因
- 生成时间
- 处理状态
- 处理人
- 处理时间

### 5.4 咨询预约表（ConsultationAppointments）
- 学生ID
- 咨询师ID
- 预约时间
- 预约状态
- 创建时间

### 5.5 咨询记录表（ConsultationRecords）
- 学生ID
- 咨询师ID
- 咨询时间
- 咨询内容
- 咨询效果评估
- 创建时间
